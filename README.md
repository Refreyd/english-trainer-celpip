# english-trainer-celpip

Простий офлайн-тренажер для CELPIP: речення з пропуском → вписуєш слово → миттєва перевірка + пояснення «чому». Прийменники, артиклі, форми дієслів.

**Працює на телефоні й компі, без додатків, без інтернету.** Шрифти системні (нічого не вантажиться ззовні).

---

## Файли

| Файл | Що це | Чіпати? |
|---|---|---|
| `index.html` | сам тренажер | **ні** — написаний раз назавжди |
| `words.js` | список карток (слова + речення + пояснення) | **так** — це єдине що ти редагуєш/заміняєш |
| `README.md` | ця інструкція | — |

Тренажер сам читає `words.js`. Щоб оновити слова — заміняєш **тільки** `words.js`, `index.html` не торкаєшся.

## Як запустити онлайн (GitHub Pages, безкоштовно)

1. Залий усі три файли в корінь репозиторію (репо має бути **публічний**).
2. **Settings → Pages → Build and deployment → Source: Deploy from a branch**.
3. Branch: `main`, папка: `/ (root)` → **Save**.
4. Через ~хвилину сайт буде тут:
   **https://refreyd.github.io/english-trainer-celpip/**
5. Відкрий це посилання на телефоні → меню браузера → **Додати на головний екран** (буде як застосунок).

Можна й без хостингу: завантажити `index.html` + `words.js` в одну папку та відкрити `index.html` у браузері (працює локально, бо дані в `.js`, а не `.json`).

## Як додати / замінити слова

**Найпростіше:** попроси Claude згенерувати новий `words.js` (промпт нижче) → відкрий `words.js` на GitHub → олівець (Edit) → встав новий вміст → **Commit**. Готово, сайт оновиться сам.

**Вручну** одне слово — додай об'єкт у список у `words.js`:

```js
{ en:"for", uk:"відповідальний за", cat:"prep",
  q:["The driver is responsible ","the accident."],
  why:"responsible завжди for, не to." },
```

## Формат картки (специфікація)

```js
{ en, uk, cat, q, why, warn }
```

- **en** — правильна відповідь (слово в пропуск).
  - «артикль не потрібен» → `"-"`
  - кілька прийнятних варіантів → масив: `["topics","subjects"]`
- **uk** — українська підказка (значення), показується над реченням.
- **cat** — `"prep"` (прийменник) · `"art"` (артикль a/an/the/-) · `"verb"` (форма дієслова). Визначає фільтр і тег.
- **q** — речення, РОЗБИТЕ на `[до_пропуску, після_пропуску]`. Пропуск стоїть між ними. Відповідь у `q` НЕ включати.
- **why** — короткий рядок українською: правило/пояснення. Показується після відповіді.
- **warn** *(необов'язково)* — `true`, якщо хочеш позначити картку тегом ⚠.

Перевірка відповіді не чутлива до регістру й зайвих пробілів. Для артиклів порожнє поле / `-` / `none` / `x` рахуються як «нічого не треба».

## Промпт для генерації нового `words.js`

Скопіюй це Claude (працює з будь-яким, навіть без пам'яті про мене):

> Згенеруй файл `words.js` для мого CELPIP cloze-тренажера. Виведи ЛИШЕ вміст файлу, без коментарів довкола.
>
> Структура:
> ```js
> window.WORDS = [
>   { en:"<відповідь>", uk:"<укр. підказка>", cat:"prep|art|verb",
>     q:["<текст до пропуску>","<текст після пропуску>"],
>     why:"<коротке пояснення українською>" },
>   // ...
> ];
> ```
> Правила:
> - `en` — слово, що йде в пропуск. Для «артикль не потрібен» → `"-"`. Для синонімів → масив, напр. `["topics","subjects"]`.
> - `cat`: `"prep"` прийменник, `"art"` артикль (a/an/the/-), `"verb"` форма дієслова.
> - `q`: розбий речення на `[до, після]` — пропуск між ними. Відповідь у `q` НЕ включай.
> - `why`: один короткий рядок українською з правилом.
> - `uk`: коротке значення українською.
>
> Ось слова / помилки, які треба перетворити на картки:
> [ВСТАВ СВІЙ СПИСОК — напр. «talent for, responsible for, in my opinion, advice (незлічуване)…» або просто речення зі своїх текстів]

Якщо просто вставиш список англійських колокацій або своїх речень — цього достатньо, Claude доробить решту.

## Примітки

- Нічого не зберігається на сервер, нема трекінгу, нема зовнішніх запитів.
- Прогрес між сесіями НЕ зберігається (кожен запуск — свіжий перемішаний раунд). Це навмисно просто.
- Хочеш більше карток / нову категорію / збереження прогресу — попроси Claude, він редагує під цей самий формат.
